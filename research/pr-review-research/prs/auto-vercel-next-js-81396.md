`🚀 1`  ·  [link](https://github.com/vercel/next.js/pull/81396#issuecomment-3140946509)

> > can easily make a new `next typegen` command
> 
> I would appreciate this a lot to be able to run in turborepo as a dependency of type checking and typed linting 
> 
> Right now we have `eslint.ignoreDuringBuilds` in our `next.config.js` because it takes a very time—instead we parallelize linting with the second half of the build, using turborepo:
> 1. `next build --experimental-build-mode compile` runs first, to generate the route types
> 2. in parallel: `eslint` and `next build --experimental-build-mode generate` run
> 
> we can’t run eslint before `next build --experimental-build-mode compile` because typed linting depends on the types generated by typedRoutes.
> 
> If there were a (fast) `next typegen` command, we could start linting earlier in the build and achieve better parallelization
> 
> Agree it wouldn’t need to be blocking for the initial implementation here, but a `next typegen` entry point would be useful for us!
